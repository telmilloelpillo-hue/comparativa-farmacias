🔄 Flujo de la nueva ruta POST /fetch_albaran

# 1. Recibe número de albarán del frontend
numero = request.json["numero"]

# 2. Detecta proveedor por prefijo
proveedor = detect_provider(numero) # "hefame" | "bida" | "unknown"

# 3a. Hefame o BIDA → portal session
if proveedor != "unknown":
  lineas = SessionManager(proveedor).fetch_albaran(numero)
  return jsonify(lineas) # misma estructura que leer_factura

# 3b. Lab externo → OCR pipeline existente (si vino foto)
else:
  return jsonify({"fallback": "ocr", "numero": numero})

📷 Frontend — Tab "Escanear albarán"

1. ZXing-js carga desde CDN (WebAssembly, sin instalación)
2. getUserMedia abre cámara → frames a ZXing cada 300ms
3. Al detectar barcode: beep + vibración, cámara se para
4. Muestra número + proveedor detectado (Hefame / BIDA / Lab externo)
5. Botón "Buscar albarán" → llama a POST /fetch_albaran
6. Tab "Subir archivo" → comportamiento actual intacto
7. Input manual siempre visible debajo del visor como fallback

🔐 SessionManager (portal_session.py)

• Singleton por proveedor (una instancia Hefame, una BIDA)
• Credenciales desde .env (HEFAME_USER / HEFAME_PASS)
• Cookie cacheada en memoria → <1s por petición si válida
• Auto re-login si respuesta 401/redirect a login
• Fetch albarán: intenta descarga PDF directa primero → parsea HTML si no hay PDF
• Si el portal no está disponible → lanza excepción → app.py hace fallback a OCR

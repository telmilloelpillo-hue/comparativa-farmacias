Con el número de albarán... ¿de dónde vienen los datos?

El código de barras lineal solo tiene un número. Para generar el PDF necesitamos los datos de los productos en algún sitio.

A

Portal web del proveedor

Hefame, BIDA, Cofares... tienen portales web donde puedes buscar un albarán por número. La app haría login con tus credenciales y descargaría el albarán automáticamente.

Ventajas

  • Datos 100% correctos, sin OCR
  • PDF oficial del proveedor

Inconvenientes

  • Requiere credenciales por proveedor
  • Los portales pueden bloquear scraping
  • Depende de acceso a internet
B

Escaneamos CN de cada producto uno a uno

Como hacen los programas de farmacia: escanas el código de barras de cada caja individual recibida. Cada escaneo añade una línea a la tabla. Al final generas el PDF del albarán reconstruido.

Ventajas

  • Sin depender de portales externos
  • Confirmas físicamente lo recibido
  • CN → nombre del producto conocido

Inconvenientes

  • Hay que escanear cada caja manualmente
  • Lento si el pedido es grande
C

Foto del albarán + código de barras como ancla

Se sigue procesando la foto con OCR/IA como ahora, pero el código de barras del documento sirve para identificar unívocamente el albarán, evitar duplicados, y etiquetar el resultado.

Ventajas

  • Mejora el flujo actual sin cambios radicales
  • El código actúa como ID de referencia

Inconvenientes

  • El OCR sigue siendo la fuente de datos
  • No elimina errores de lectura
Contexto

En Hefame y BIDA el código de barras del documento (albarán) suele ser el número de albarán. El código de barras de cada caja es el CN (Código Nacional) del medicamento — distinto.
